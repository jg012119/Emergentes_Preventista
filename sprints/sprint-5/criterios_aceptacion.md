# ✅ Sprint 5 — Criterios de Aceptación

## Encuestas de satisfacción

- [ ] Se diseñó una encuesta para clientes con al menos 15 preguntas (Likert, texto libre, opción múltiple).
- [ ] Se diseñó una encuesta para empresa AJE con al menos 10 preguntas.
- [ ] Se ejecutaron al menos 5 sesiones de prueba con usuarios de rol cliente.
- [ ] Se ejecutaron al menos 2 sesiones de prueba con usuarios de rol empresa AJE.
- [ ] Todas las respuestas fueron recopiladas y tabuladas.
- [ ] Las encuestas se realizaron con formularios digitales (Google Forms o similar).

---

## Métricas de uso

- [ ] Se registró la cantidad total de pedidos creados durante las pruebas.
- [ ] Se registró el tiempo promedio para crear un pedido por cada método (manual, texto, voz).
- [ ] Se registró la tasa de éxito del NLP (≥ 80% de pedidos correctamente interpretados).
- [ ] Se registró la tasa de éxito del reconocimiento de voz.
- [ ] Se registró la cantidad de correcciones manuales necesarias.
- [ ] Se compararon los tiempos entre los 3 métodos de entrada.

---

## Análisis de aceptación

- [ ] Se tabularon los promedios de satisfacción por pregunta (escala 1-5).
- [ ] El promedio general de satisfacción de clientes es ≥ 3.5/5.
- [ ] El promedio general de satisfacción de empresa AJE es ≥ 3.5/5.
- [ ] Se calculó el Net Promoter Score (NPS).
- [ ] Se crearon al menos 4 gráficos de visualización de resultados.
- [ ] Se identificaron las 3 principales fortalezas del producto.
- [ ] Se identificaron las 3 principales áreas de mejora.
- [ ] Se analizaron las respuestas de texto libre.

---

## Reporte de efectividad NLP/voz

- [ ] Se documentó la precisión del NLP en una tabla resumen.
- [ ] Se documentó la precisión del reconocimiento de voz en una tabla resumen.
- [ ] Se analizaron los tipos de errores más frecuentes.
- [ ] Se creó un gráfico comparativo (manual vs NLP vs voz).
- [ ] Se documentaron recomendaciones para futuras versiones.

---

## Informe ejecutivo

- [ ] El informe ejecutivo tiene entre 1-2 páginas.
- [ ] Incluye resumen del proyecto.
- [ ] Incluye tabla de objetivos cumplidos vs no cumplidos.
- [ ] Incluye métricas clave de aceptación.
- [ ] Incluye resultados principales de encuestas.
- [ ] Incluye análisis de viabilidad.
- [ ] Incluye recomendaciones para fases futuras.
- [ ] Incluye lecciones aprendidas.
- [ ] Fue revisado y aprobado por todo el equipo.

---

## Presentación final

- [ ] Presentación de diapositivas con mínimo 20 slides.
- [ ] Video demostrativo de 3-5 minutos del sistema funcionando.
- [ ] El video muestra flujo de pedido por texto.
- [ ] El video muestra flujo de pedido por voz.
- [ ] El video muestra panel AJE con gestión de pedidos.
- [ ] Se realizaron al menos 2 ensayos de la presentación.
- [ ] Se prepararon respuestas a preguntas frecuentes.

---

## Documentación técnica

- [ ] README.md actualizado con estado final del proyecto.
- [ ] Instrucciones de instalación y despliegue documentadas.
- [ ] API del backend documentada (endpoints, parámetros, respuestas).
- [ ] Motor NLP documentado.
- [ ] Integración de voz documentada.
- [ ] Diagrama de arquitectura final creado.
- [ ] Diagrama ER de base de datos creado.

---

## Criterio global de completitud

> **El Sprint 5 se considera completo cuando se tienen los reportes completos de aceptación del producto basados en encuestas reales a usuarios de prueba, las métricas de uso están documentadas, el informe ejecutivo está aprobado, la presentación final y el video demostrativo están listos, y toda la documentación técnica está actualizada. El proyecto debe estar listo para defensa ante la stakeholder.**
