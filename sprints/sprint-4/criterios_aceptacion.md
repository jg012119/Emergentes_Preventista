# ✅ Sprint 4 — Criterios de Aceptación

## Pruebas unitarias

- [ ] Al menos 10 tests unitarios escritos para los endpoints principales del backend.
- [ ] Todos los tests pasan exitosamente.
- [ ] Los tests cubren casos exitosos y de error.
- [ ] Los resultados están documentados.

---

## Pruebas de integración

- [ ] El flujo de registro → login → tienda → pedido manual → confirmar funciona end-to-end.
- [ ] El flujo de chat escrito → NLP → extracto → confirmar funciona end-to-end.
- [ ] El flujo de voz → texto → NLP → extracto → confirmar funciona end-to-end.
- [ ] El flujo de pedido → AJE cambia estado → notificación funciona end-to-end.
- [ ] La integración app ↔ backend ↔ base de datos es estable.
- [ ] La integración panel ↔ backend ↔ base de datos es estable.

---

## Pruebas del NLP

- [ ] Se probaron al menos 10 frases diferentes de pedidos.
- [ ] La tasa de precisión de identificación de productos es ≥ 80%.
- [ ] El sistema maneja correctamente pedidos con errores ortográficos comunes.
- [ ] El sistema maneja correctamente mensajes que no son pedidos.
- [ ] Los resultados de precisión están documentados.

---

## Pruebas de reconocimiento de voz

- [ ] Se probaron al menos 5 frases en ambiente silencioso.
- [ ] Se probaron al menos 5 frases en ambiente con ruido.
- [ ] La tasa de precisión en ambiente silencioso es ≥ 70%.
- [ ] La corrección manual funciona correctamente.
- [ ] Los resultados están documentados.

---

## Pruebas de usabilidad

- [ ] Se evaluaron al menos 8 flujos principales de la app.
- [ ] Se solicitó retroalimentación a al menos 3 usuarios de prueba.
- [ ] Los hallazgos de usabilidad están documentados.
- [ ] Se identificaron y priorizaron mejoras necesarias.

---

## Pruebas de aceptación

- [ ] Todos los 20 criterios de aceptación del proyecto fueron verificados.
- [ ] Al menos el 90% de los criterios se cumplen satisfactoriamente.
- [ ] Los criterios no cumplidos tienen justificación documentada.
- [ ] El checklist de criterios está completo y firmado.

---

## Corrección de errores

- [ ] Todos los errores "Críticos" fueron corregidos.
- [ ] Todos los errores "Altos" fueron corregidos.
- [ ] Los errores "Medios" fueron corregidos según el tiempo disponible.
- [ ] Se ejecutaron pruebas de regresión después de cada corrección.
- [ ] El registro de errores (bug tracker) está completo y actualizado.

---

## Documentación

- [ ] Documento de pruebas unitarias completado.
- [ ] Documento de pruebas de integración completado.
- [ ] Documento de pruebas de NLP completado.
- [ ] Documento de pruebas de voz completado.
- [ ] Documento de pruebas de usabilidad completado.
- [ ] Documento de pruebas de aceptación completado.
- [ ] Registro de errores completado.

---

## Criterio global de completitud

> **El Sprint 4 se considera completo cuando todos los tipos de pruebas se ejecutaron, los errores críticos y altos fueron corregidos, las pruebas de regresión pasaron, la tasa de precisión del NLP es ≥ 80%, y toda la documentación de pruebas está completa.**
