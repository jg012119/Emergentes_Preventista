Recorrido del Pedido:

  1. La tienda dicta o escribe su pedido por voz o chat.
  2. El sistema reconoce de forma automatizada productos y cantidades.
  3. Se genera y presenta un borrador del pedido en pantalla.
  4. El usuario revisa y ajusta la información de la orden.
  5. El pedido se confirma y queda registrado en la consola de AJE.

Casos de Prueba Interactivos:

Una solución con beneficios para cada actor del proceso

El sistema conecta la necesidad de la tienda con la operación comercial de AJE.

A

Para la Tienda

  • - Reposición inmediata: Pedidos rápidos sin esperar al preventista.
  • - Precios claros: Transparencia total de montos en pantalla.
  • - Autonomía: Consulta del estado de entrega desde el chat.
B

Para el Preventista

  • - Menos burocracia: Menor tiempo en transcripción de pedidos rutinarios.
  • - Mejor preparación: Contexto del cliente antes de la visita física.
  • - Foco comercial: Más tiempo para negociación y exhibición en tienda.
C

Para AJE

  • - Centralización: Registro de pedidos directo en la base de datos.
  • - Trazabilidad: Monitoreo logístico de despachos desde la consola.
  • - Demanda: Datos de compra útiles para planificar distribución.

Probamos el sistema, identificamos fricciones y aplicamos mejoras

La evaluación permitió medir la facilidad de uso y corregir problemas antes de proponer un piloto operativo.

Resultados de Usabilidad

Evaluación académica con 33 participantes: 89.2% de pedidos completados exitosamente, con un tiempo promedio de registro de 92.4 segundos.

Español Nativo (Bolivia) 79.4 / 100 (Muestra local)
Español Limitado (EEUU Muestra) 36.6 / 100 (Fricción por acento)
Promedio SUS Global 58.6 / 100 (Oportunidad de mejora)
*Nota de alcance: La muestra corresponde a una validación académica de usabilidad. Los resultados no representan todavía una validación comercial con la red de tiendas de AJE.

Arquitectura preparada para validación y crecimiento

Resumen técnico de la infraestructura modular para asegurar robustez, velocidad y seguridad.

Componentes del Sistema

Aplicación Móvil

Desarrollada en React Native / Expo, con interfaz adaptativa para registro de pedidos y visualización de catálogo.

Backend en FastAPI

Pipeline de procesamiento veloz en Python, implementando enrutamiento y orquestación de datos de catálogo.

Base de Datos en Supabase

Centralización de información con PostgreSQL, seguridad mediante RLS (Row Level Security) y autenticación JWT.

Estrategia Conversacional

-
Procesamiento Híbrido:

Capa de reglas iniciales para reconocimiento directo de SKUs, y modelos de lenguaje local para expresiones coloquiales.

-
Seguridad y Privacidad:

Los datos de las transacciones y clientes permanecen centralizados y encriptados en la infraestructura local.

-
Escalabilidad:

Diseño modular que facilita la integración con sistemas ERP de facturación o inventario existentes.
